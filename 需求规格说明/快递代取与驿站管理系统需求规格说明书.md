# 快递代取与驿站管理系统智慧末端驿站（快递服务站）管理需求规格说明书

## 1. 核心业务流澄清与概述

### 1.1 系统定位

本系统是为**校园/社区快递驿站**量身定制的全流程管理平台。系统以**“包裹”**为数据核心，以**“入库→通知→出库”**为生命主轴。代取服务仅作为驿站业务的一个增值插件，用于处理用户不便亲临的场景。

### 1.2 业务真实现状

1. **上游**：快递员/配送员将大批包裹送至驿站。
2. **入库上架**：驿站管理员扫码或手动录入单号，系统自动关联取件码，并分配货架编号，包裹状态变为“**待取件**”。
3. **通知触达**：系统通过微信服务通知/短信，向收件人推送“包裹已到站 + 取件码 + 货架号”。
4. **常规出库（95%场景）**：用户到店，凭取件码或扫码验证身份，管理员确认出库，状态变更为“**已取件**”。
5. **异常处理**：超时未取，系统标记为“**滞留件**”，触发催取或退件流程。
6. **小众场景（<5%）**：用户无法自取 → 发布“代取悬赏” → 跑腿人员抢单 → 持电子授权码代取 → 确认送达。

## 2. 用户角色重新定义

| 角色                         | 身份          | 核心职责                                                   |
| :--------------------------- | :------------ | :--------------------------------------------------------- |
| **驿站管理员**         | 驿站老板/店员 | 包裹入库上架、货架管理、出库核销、异常件处理、营收对账     |
| **普通收件人**         | 小区业主/学生 | 接收取件通知、查询取件码、到店自取、发起代取需求、确认收货 |

| **系统管理员**         | 平台运营方    | 系统配置、用户权限管理、全局数据监控                       |

## 3. 小程序端功能需求

### 3.1 收件人（用户端）

**3.1.1 包裹看板（首页）**

- **待取件列表**：展示当前驿站内所有本人名下的“待取件”包裹。
- **关键信息**：快递公司、货架编号、入库时间、滞留天数（超过3天变红预警）。
- **一键导航**：点击跳转地图导航至驿站。

**3.1.2 取件码查询（核心）**

- 展示包裹对应的**动态取件码**（支持一维码/二维码，便于自助扫码出库）。
- **安全防护**：取件码在包裹出库后自动失效隐藏。

**3.1.3 自主出库（创新）**

- 用户点击“我要取件”，系统生成动态核销码。
- 管理员扫描用户手机核销码，或用户扫描驿站固定的核销二维码，完成自助出库。

**3.1.4 代取发起（辅助功能）**

- 针对“待取件”包裹，点击“**找人代取**”。
- 设置代取悬赏金额、取件截止时间。
- 发布后，订单进入“代取大厅”，等待跑腿员接单。
- **流程**：跑腿员接单 → 获得一次性授权取件码 → 至驿站取件 → 送达后收件人确认 → 赏金结算。

**3.1.5 消息中心**

- 接收入库通知（新包裹到达）。
- 接收滞留提醒（超时未取）。
- 接收代取订单状态流转通知。

---

### 3.2 跑腿员端（接单方）

**3.2.1 代取任务大厅**

- 展示驿站内用户发布的“待接单”代取任务。
- 按距离、悬赏金额、取件时效排序。

**3.2.2 接单与配送**

- 接单后获取**临时取件码**（仅限此包裹）。
- 到驿站取件时出示临时码，管理员核验后放行。
- 配送完成点击“已送达”，等待用户确认。

**3.2.3 我的收益**

- 查看已完成代取订单的累计收益、可提现余额。

## 4. 管理后台功能需求

### 4.1 包裹入库管理

- **极速入库**：支持扫码枪扫描快递单号（菜鸟/丰巢面单标准条码），自动识别快递公司。
- **手动录入**：手动输入单号、选择快递公司、输入收件人手机号后四位（自动匹配系统内用户，若不存在则注册为匿名用户）。
- **货架分配**：入库时手动/自动分配货架编码（如 A-01, B-12），支持批量选中包裹统一分配货架。
- **批量导入**：支持 Excel 表格批量导入大批量包裹信息（大促期间使用）。
- **入库打标**：标记“生鲜件”、“大件”、“贵重物品”等特殊标签，方便特殊管理。

### 4.2 货架与库位管理

- 可视化编辑驿站库位（几排几列）。
- 实时显示每个货架的**包裹占用数量**。
- 支持“滞销货架”清理功能（超过7天未取的包裹自动移入滞留区）。

### 4.3 出库核销管理

- **扫码出库**：管理员扫描用户出示的取件码/包裹单号，一键确认出库。
- **手动出库**：输入手机号后四位或取件码，手动点击出库。
- **批量出库**：针对同一个收件人的多个包裹，支持合并一键取走。

### 4.4 订单/包裹管理

- **包裹列表**：展示所有历史及当前包裹，状态筛选（待取件、已取件、滞留件、已退件）。
- **包裹详情**：查看完整物流轨迹（入库时间、取件时间、取件人身份记录）。
- **异常处理**：标记“错拿”、“破损”、“退件”，并记录处理日志。
- **代取记录**：关联查看该包裹的代取任务详情（若有）。

### 4.5 用户管理

- **收件人管理**：按手机号/姓名检索，查看该用户所有历史包裹取件记录。
- **黑名单管理**：恶意错拿、逾期不取的用户可加入黑名单，限制其下次接收快递通知。
- **跑腿员资质审核**：审核跑腿员的实名信息，代取能力认证。

### 4.6 营收统计

- **包裹流量看板**：今日入库量、今日出库量、当前滞留量、月度趋势折线图。
- **代取财务报表**：平台代取悬赏抽成统计、跑腿员结算清单。
- **快递公司对账**：按快递公司维度统计派件量，便于向快递员收取派件代管费。

## 5. 本地服务端功能需求

### 5.1 包裹生命周期管控

- **状态机**：`已入库待取` ↔ `取件中（代取授权）` ↔ `已取件（完成）` ↔ `滞留预警` ↔ `已退件`。
- **取件码生成规则**：基于包裹ID+用户手机尾号加密生成6位数字，有效期为包裹出库前。

### 5.2 超时滞留自动处理

- **时间阈值**：入库后 **24小时** 推送一次微信催取通知；**72小时** 自动标记“超时滞留”，并发送人工提醒给管理员；**7天** 自动发起“是否退件”确认流程，用户不回复则自动退回。

### 5.3 用户身份校验（防错拿）

- **双重校验**：取件时，除了取件码，后台悄悄比对取件人的**地理位置**（是否在驿站500米范围内），若不在范围内且是代取，强制标记为“需人工核验”。
- **频次校验**：同一用户在短时间内（5分钟）连续核销不同包裹，触发“批量取件”风控提示。

### 5.4 数据存储与备份

- 实时记录**操作日志**（谁、何时、用什么方式取走了哪个包裹）。
- 每日凌晨2点自动备份前一日包裹流水数据。

## 6. 可选创新方向

### 6.1 智能语音播报（提效）

- 管理员入库扫码时，系统自动语音播报“**申通，张三，A区3号**”，无需频繁看屏幕，提升入库效率。

### 6.2 驿站自助出库终端（IoT模拟）

- 模拟连接蓝牙扫码枪或自助出库机。
- 用户将手机取件码对准自助机扫描 → 自助机识别后自动点亮对应**货架LED小灯**（模拟功能），引导用户找到包裹。

### 6.3 信用分双向约束

- **针对用户**：逾期超过3天不取，每次扣除信用分；信用分过低无法发起“代取”请求（倒逼及时取件）。
- **针对跑腿员**：接单后超时未取/配送丢件，扣除信用保证金。

### 6.4 代取智能运费计算（极简版）

- 代取费不再基于距离（驿站位置固定），而是基于**包裹重量**（大件加价）+ **时效要求**（加急加价）+ **天气因素**（雨雪天气补贴）。

## 8. 核心业务流程梳理

### 流程一：包裹入库与用户自取

> 快递员卸货 → 管理员扫描单号 → 系统分配取件码+货架号 → 用户微信收到通知 → 用户到店出示取件码 → 管理员扫码核销/自助扫码机器（预留api不做自助扫码机器） → 包裹出库（结束）。

### 流程二：代取场景

> 用户收到通知但不便取件 → 在小程序端点击“找代取”并悬赏 → 跑腿员在大厅抢单 → 跑腿员获得临时授权码 → 到店取件（管理员核验授权码） → 跑腿员配送到用户手中 → 用户确认收货 → 赏金结算。

## 9. 技术非功能性要求

- 小程序端极速加载，针对弱网环境做优化（驿站网络常不好）。
- 扫码识别响应时间 < 300ms。
- 后台支持高并发入库（应对双11大促瞬间涌入）。
