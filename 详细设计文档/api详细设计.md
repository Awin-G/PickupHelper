## 1. 通用约定

### 1.1 基础规范

- 接口基础前缀：`/api/v1`
- 传输协议：HTTPS（生产环境强制），HTTP 仅用于本地开发
- 数据格式：`Content-Type: application/json; charset=utf-8`
- 字符编码：UTF-8
- 时间格式：所有时间字段使用 ISO 8601（`YYYY-MM-DDTHH:mm:ss+08:00`），时区为东八区（Asia/Shanghai）
- 货币单位：人民币（CNY），单位「元」，保留两位小数（如 `5.00`）
- 路径参数：使用小写连字符（kebab-case），如 `/proxy/my-orders`
- 字段命名：请求/响应 JSON 字段统一使用 `snake_case`
- 幂等性：所有 `POST` 写接口支持可选 `Idempotency-Key` 请求头，30 分钟内同 key 同请求体视为重复请求直接返回首次结果

### 1.2 鉴权方式

- 认证方案：JWT（Access Token + Refresh Token）
- 携带方式：HTTP Header
  ```
  Authorization: Bearer <access_token>
  ```
- Token 有效期：
  - Access Token：2 小时
  - Refresh Token：7 天
- 公开接口（`/auth/send-code`、`/auth/login`、`/auth/refresh`）无需鉴权
- 已登录接口缺失或携带非法 Token 返回 `401`，权限不足返回 `403`
- 鉴权中间件解析 Token 后向下游注入 `X-User-Id`、`X-User-Type`、`X-Station-Id`、`X-Role` 请求头（仅内部传递，不可由客户端伪造）

### 1.3 统一响应结构

```json
{
  "code": 0,
  "msg": "success",
  "data": {},
  "trace_id": "a1b2c3d4e5f6"
}
```

| 字段        | 类型    | 必填 | 说明                                                  |
| ----------- | ------- | ---- | ----------------------------------------------------- |
| `code`      | integer | 是   | 业务状态码，`0` 表示成功，非 `0` 表示失败             |
| `msg`       | string  | 是   | 描述信息，成功为 `success`，失败为可读的错误描述      |
| `data`      | object  | 否   | 业务数据，失败时可为 `null`                           |
| `trace_id`  | string  | 是   | 链路追踪 ID，用于问题定位，与请求日志一一对应         |

HTTP 状态码与 `code` 配合使用：

| HTTP 状态码 | 含义                                       |
| ----------- | ------------------------------------------ |
| 200         | 请求被正确处理（业务成功或业务失败均返回） |
| 400         | 请求参数格式错误、缺必填项                 |
| 401         | 未认证或 Token 失效                        |
| 403         | 已认证但无权限访问该资源                   |
| 404         | 资源不存在                                 |
| 409         | 资源状态冲突（如重复入库、状态非法流转）   |
| 429         | 请求频率超限                               |
| 500         | 服务端内部错误                             |

### 1.4 分页约定

- 请求参数（Query）：

| 字段         | 类型    | 必填 | 默认 | 说明                                            |
| ------------ | ------- | ---- | ---- | ----------------------------------------------- |
| `page`       | integer | 否   | 1    | 页码，从 1 开始                                 |
| `page_size`  | integer | 否   | 20   | 每页条数，最大 100                              |
| `sort`       | string  | 否   | -    | 排序字段，格式 `field` 或 `-field`（降序）      |
| `keyword`    | string  | 否   | -    | 通用搜索关键字                                  |

- 响应结构：

```json
{
  "list": [],
  "total": 0,
  "page": 1,
  "page_size": 20
}
```

| 字段         | 类型    | 说明           |
| ------------ | ------- | -------------- |
| `list`       | array   | 当前页数据列表 |
| `total`      | integer | 总记录数       |
| `page`       | integer | 当前页码       |
| `page_size`  | integer | 每页条数       |

### 1.5 通用请求头

| Header                | 必填 | 说明                                                    |
| --------------------- | ---- | ------------------------------------------------------- |
| `Authorization`       | 视接口而定 | 鉴权 Token                                       |
| `Content-Type`        | 是   | `application/json; charset=utf-8`                       |
| `X-Client-Type`       | 是   | 客户端类型：`miniapp` / `admin` / `iot`                 |
| `X-Client-Version`    | 是   | 客户端版本号，如 `1.2.0`                                |
| `X-Device-Id`         | 否   | 设备唯一标识，用于风控                                  |
| `X-Geo-Lat`           | 否   | 客户端纬度（取件、自取出库等接口必填）                  |
| `X-Geo-Lng`           | 否   | 客户端经度（取件、自取出库等接口必填）                  |
| `Idempotency-Key`     | 否   | 幂等键，写操作建议携带                                  |
| `Accept-Language`     | 否   | 多语言预留，默认 `zh-CN`                                |

### 1.6 字段类型与语义

| 类型      | 说明                                                                |
| --------- | ------------------------------------------------------------------- |
| `string`  | 字符串，所有手机号、单号等需明确最大长度                            |
| `integer` | 整数，枚举值使用 `tinyint` 语义                                     |
| `number`  | 浮点数，金额、经纬度等                                              |
| `boolean` | 布尔值，仅 `true` / `false`                                         |
| `datetime`| ISO 8601 字符串                                                     |
| `array`   | 数组                                                                |
| `object`  | 嵌套对象                                                            |

字段必填标注：`必填` / `可选` / `只读`（响应中出现，请求不可设置）/ `忽略`（请求不生效）。

### 1.7 错误码规范

错误码全局唯一，分为通用错误码与模块业务错误码。范围为：

| 范围        | 说明             |
| ----------- | ---------------- |
| `0`         | 成功             |
| `10001~10099` | 通用错误       |
| `10101~10199` | 用户模块错误   |
| `10201~10299` | 包裹模块错误   |
| `10301~10399` | 取件模块错误   |
| `10401~10499` | 代取模块错误   |
| `10501~10599` | 货架模块错误   |
| `10601~10699` | 通知模块错误   |
| `99999`       | 未知错误       |

### 1.8 通用业务错误码

| code  | HTTP | 说明                       |
| ----- | ---- | -------------------------- |
| 0     | 200  | 成功                       |
| 10001 | 400  | 请求参数错误               |
| 10002 | 401  | 未登录或 Token 失效        |
| 10003 | 403  | 无权限访问                 |
| 10004 | 404  | 资源不存在                 |
| 10005 | 409  | 资源状态冲突               |
| 10006 | 429  | 请求频率超限               |
| 10007 | 413  | 上传文件过大               |
| 10008 | 415  | 不支持的媒体类型           |
| 10009 | 500  | 服务端内部错误             |
| 10010 | 400  | Idempotency-Key 重复请求   |
| 99999 | 500  | 未知错误                   |

## 2. 用户模块（User / Auth）

### 2.1 发送手机验证码

- 方法：`POST`
- 路径：`/auth/send-code`
- 权限：公开
- 频率限制：同一手机号 60s 内 1 次，同一 IP 10 分钟内 10 次

**请求体**

| 字段      | 类型   | 必填 | 说明                                       |
| --------- | ------ | ---- | ------------------------------------------ |
| `phone`   | string | 是   | 手机号，正则 `^1[3-9]\d{9}$`               |
| `scene`   | string | 否   | 场景：`login` / `runner_apply`，默认 `login` |

**请求示例**

```json
{ "phone": "13800138000", "scene": "login" }
```

**响应体**

| 字段        | 类型   | 说明                              |
| ----------- | ------ | --------------------------------- |
| `expire_in` | integer | 验证码有效期（秒），固定 300      |

**响应示例**

```json
{ "code": 0, "msg": "success", "data": { "expire_in": 300 }, "trace_id": "..." }
```

**业务错误码**

| code  | 说明                       |
| ----- | -------------------------- |
| 10101 | 手机号格式错误             |
| 10102 | 发送频率超限               |
| 10103 | 短信通道异常               |
| 10104 | 手机号在黑名单中           |

---

### 2.2 手机号验证码登录/注册

- 方法：`POST`
- 路径：`/auth/login`
- 权限：公开

**请求体**

| 字段     | 类型   | 必填 | 说明                       |
| -------- | ------ | ---- | -------------------------- |
| `phone`  | string | 是   | 手机号                     |
| `code`   | string | 是   | 6 位短信验证码             |
| `openid` | string | 否   | 微信 openid，小程序必传    |

**响应体**

| 字段           | 类型    | 说明                                  |
| -------------- | ------- | ------------------------------------- |
| `access_token` | string  | 访问令牌，2h 有效                     |
| `refresh_token`| string  | 刷新令牌，7d 有效                     |
| `expires_in`   | integer | access_token 有效期（秒）             |
| `user`         | object  | 用户基本信息，结构见 [2.4](#24-获取当前用户信息) |

**业务错误码**

| code  | 说明                       |
| ----- | -------------------------- |
| 10111 | 验证码错误或已过期         |
| 10112 | 验证码已使用               |
| 10113 | 用户已被拉黑               |

- 权限：已登录（携带过期 access_token，refresh_token 仍有效）

**请求体**

| 字段            | 类型   | 必填 | 说明         |
| --------------- | ------ | ---- | ------------ |
| `refresh_token` | string | 是   | 刷新令牌     |

**响应体**

| 字段            | 类型    | 说明                |
| --------------- | ------- | ------------------- |
| `access_token`  | string  | 新访问令牌          |
| `expires_in`    | integer | 有效期（秒）        |

**业务错误码**

| code  | 说明                  |
| ----- | --------------------- |
| 10121 | refresh_token 无效    |
| 10122 | refresh_token 已过期  |

- 方法：`GET`
- 路径：`/user/info`
- 权限：已登录

**响应体**

| 字段             | 类型    | 说明                                                  |
| ---------------- | ------- | ----------------------------------------------------- |
| `id`             | integer | 用户 ID                                               |
| `phone`          | string  | 手机号（脱敏：`138****0000`）                         |

| `user_type`      | integer | 1-普通收件人，2-跑腿员                                |

| `station_id`     | integer | 所属驿站（管理员字段，普通用户为 `null`）             |
| `created_at`     | datetime| 注册时间                                              |

**响应体**

| 字段           | 类型    | 说明                       |

### 2.8 审核跑腿员申请

- 方法：`PUT`
- 路径：`/user/runner/applications/{id}/audit`
- 权限：管理员

**路径参数**

| 字段 | 类型    | 说明        |
| ---- | ------- | ----------- |
| `id` | integer | 申请单 ID   |

**请求体**

| 字段            | 类型    | 必填 | 说明                              |
| --------------- | ------- | ---- | --------------------------------- |
| `action`        | string  | 是   | `approve` / `reject`              |
| `audit_remark`  | string  | 否   | 审核备注，最长 255 字符           |

**响应体**：返回更新后的申请单（结构同 [2.7](#27-查看跑腿员申请列表)）。

**业务错误码**

| code  | 说明                              |
| ----- | --------------------------------- |
| 10151 | 申请单不存在                      |
| 10152 | 申请单状态非「审核中」            |
| 10153 | `action` 取值非法                 |

### 2.9 管理端查看活跃验证码

- 方法：`GET`
- 路径：`/admin/auth/sms-codes`
- 权限：管理员
- 说明：返回所有尚未过期（TTL>0）且未被使用的短信验证码。验证码存储于 Redis，过期后由 Redis 自动删除。由于短信 API 尚为 stub 实现，此接口可用于开发/测试阶段排查已生成的验证码。

**请求体**：无

**响应体**

| 字段     | 类型   | 说明                              |
| -------- | ------ | --------------------------------- |
| `codes`  | array  | 活跃验证码列表，元素结构见下      |
| `total`  | integer| 验证码总数                        |

`codes` 元素：

| 字段         | 类型    | 说明                        |
| ------------ | ------- | --------------------------- |
| `phone`      | string  | 手机号                      |
| `code`       | string  | 6 位短信验证码              |
| `expire_in`  | integer | 剩余有效时间（秒）          |

**响应示例**

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "codes": [
      { "phone": "13800138000", "code": "123456", "expire_in": 120 },
      { "phone": "13900139000", "code": "654321", "expire_in": 45 }
    ],
    "total": 2
  },
  "trace_id": "..."
}
```

**业务错误码**：无特有错误码。鉴权失败复用通用 10002、10003。

## 3. 包裹模块（Parcels）

### 3.1 扫码/手动入库

- 方法：`POST`
- 路径：`/parcels/scan-in`
- 权限：管理员（仅本驿站）
- 风控：操作人必须归属当前 `station_id`

**请求体**

| 字段                | 类型    | 必填 | 说明                                              |
| ------------------- | ------- | ---- | ------------------------------------------------- |
| `tracking_no`       | string  | 是   | 快递单号，最长 64                                 |
| `courier_company`   | string  | 是   | 快递公司名称（与字典一致）                        |
| `receiver_phone`    | string  | 是   | 收件人手机号                                      |
| `receiver_name`     | string  | 否   | 收件人姓名                                        |
| `shelf_code`        | string  | 否   | 货架编号，留空则自动分配                          |
| `weight`            | number  | 否   | 重量，单位 kg，默认 0                             |
| `is_fragile`        | boolean | 否   | 是否易碎，默认 false                              |
| `remarks`           | string  | 否   | 备注，最长 255                                    |
| `storage_time`      | datetime| 否   | 入库时间，默认当前时间                            |

**响应体**

| 字段            | 类型    | 说明                              |
| --------------- | ------- | --------------------------------- |
| `parcel_id`     | integer | 包裹 ID                           |
| `pickup_code`   | string  | 6 位取件码                        |
| `shelf_code`    | string  | 实际分配的货架编号                |
| `storage_time`  | datetime | 入库时间                         |

**业务错误码**

| code  | 说明                                            |
| ----- | ----------------------------------------------- |
| 10201 | 快递单号已存在（同驿站内）                      |
| 10202 | 货架编号不存在或已满                            |
| 10203 | 收件人手机号格式错误                            |
| 10204 | 快递公司不在字典中                              |
| 10205 | 取件码生成冲突重试失败（极小概率）              |

### 3.2 批量导入包裹

- 方法：`POST`
- 路径：`/parcels/batch-in`
- 权限：管理员
- Content-Type：`multipart/form-data`
- 处理方式：异步。服务端校验文件格式后立即返回 `batch_id`，通过 MQ 异步逐行处理，前端可通过 `/parcels/batch-in/{batch_id}` 查询进度（该查询接口预留，本文档暂不展开）。

**请求体（form-data）**

| 字段             | 类型   | 必填 | 说明                                              |
| ---------------- | ------ | ---- | ------------------------------------------------- |
| `file`           | file   | 是   | Excel 文件（.xlsx），单文件 ≤ 5MB                 |
| `station_id`     | integer| 是   | 入库目标驿站 ID（系统管理员可指定，驿站管理员固定为本站） |

**响应体**

| 字段        | 类型    | 说明                       |
